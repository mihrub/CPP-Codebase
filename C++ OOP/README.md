---

### 1. Encapsulation

Encapsulation means binding data and functions together inside a class.

Main idea:
- data is hidden
- accessed using public functions

Why used:
- protects data
- controls how data is modified

Example concept:
- private data members
- public getters and setters

---

### 2. Abstraction

Abstraction means hiding implementation details and showing only essential features.

Why used:
- reduces complexity
- improves clarity for the user of a class

In C++ it is achieved using:
- abstract classes
- pure virtual functions

---

### 3. Inheritance

Inheritance allows one class to acquire properties and behavior of another class.

Why used:
- code reuse
- logical relationship between classes

Types commonly used:
- single inheritance
- multilevel inheritance
- hierarchical inheritance

---

### 4. Polymorphism

Polymorphism means one interface, multiple behaviors.

Two main types:

âœ” Compile-time polymorphism
- function overloading
- operator overloading

âœ” Runtime polymorphism
- function overriding using virtual functions

Why used:
- flexible and extensible design
